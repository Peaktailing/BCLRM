## 一、试剂领用

### 1.1 入口

页面文件：`pages/3_领用归还.py`
页面内通过 `st.tabs` 分为两个标签页：「试剂领用」和「试剂归还」。
本文档仅描述「试剂领用」流程，「试剂归还」见第二节。

### 1.2 搜索与筛选

1. 用户在搜索框中输入关键词（试剂名称、CAS号、编号均可）。
2. 用户可勾选「只显示可借」复选框（默认勾选）。
3. 页面调用 `query_service.filter_reagents(keyword=..., borrowable_only=...)` 进行过滤。
   - 引用模块：`business/query_service.py` → `QueryService.filter_reagents()`
   - 该方法内部调用 `services/core/reagent_bottle_service.py` → `ReagentBottleService.get_all_parsed()` 获取全部试剂瓶数据，然后在 Python 层做关键词匹配和状态过滤。
4. 过滤结果以表格形式展示，包含列：选择（复选框）、编号、名称、CAS号、剩余量、规格、纯度、状态（🟢可借/🔵已借出/🔴耗尽）、存储位置。

> **业务规则确认**：只有 `borrowable_flag = "可借"` 且 `remaining_quantity > 0` 的试剂瓶才可被借出。当前为整瓶借出模式，领用数量等于试剂瓶当前剩余量。

### 1.3 添加到领用清单

1. 用户在表格中勾选一行或多行试剂。
2. 点击「添加 N 瓶试剂到领用清单」按钮。
3. 选中的试剂被追加到 `st.session_state.borrow_cart`（购物车列表），已存在于清单中的不重复添加。
4. 页面下方展示领用清单表格，包含列：序号、编号、名称、CAS号、规格、剩余量、删除。
5. 用户可在清单中删除单条（修改删除列内容）或点击「清空清单」全部清空。

> **交互说明**：当前删除操作通过修改"删除"列内容实现（非🗑️则删除），此交互方式待后续测试优化。

### 1.4 选择领用人

1. 页面调用 `borrow_service.get_all_borrow_users()` 获取历史领用人列表。
   - 引用模块：`business/borrow_service.py` → `BorrowService.get_all_borrow_users()`
   - 该方法内部调用 `services/core/borrow_record_service.py` → `BorrowRecordService.get_all_parsed()` 获取全部领用记录，提取去重后的用户名列表并排序。
2. 用户通过下拉框选择领用人。

### 1.5 确认领用

1. 用户点击「确认领用」按钮。
2. 页面遍历领用清单中的每一瓶试剂，逐瓶调用 `borrow_service.reagent_borrow(bottle_number, user, borrow_qty)`。
   - 引用模块：`business/borrow_service.py` → `BorrowService.reagent_borrow()`
   - `borrow_qty` 取该试剂瓶的当前剩余量（即整瓶借出）。

3. **业务层 `reagent_borrow()` 内部执行以下校验和操作流程**：

   **步骤 1 — 参数校验**
   - 校验 `bottle_number` 为正整数
   - 校验 `user` 非空
   - 校验 `borrow_qty` 大于 0
   - 任一失败 → 返回 `ServiceResult.fail`

   **步骤 2 — 查询试剂瓶**
   - 调用 `reagent_bottle_service.get_by_bottle_number(bottle_number)` 查询试剂瓶
   - 引用模块：`services/core/reagent_bottle_service.py` → `ReagentBottleService.get_by_bottle_number()`
   - 未找到 → 返回失败「未找到该试剂瓶」

   **步骤 3 — 校验试剂瓶状态**
   - 检查 `borrowable_flag` 是否为「可借」
   - 非「可借」→ 返回失败「该试剂当前状态为『xxx』，不可借出」

   **步骤 4 — 校验领用数量**
   - 检查剩余量是否已知（非 None）
   - 检查 `borrow_qty <= remaining_qty`
   - 超出 → 返回失败「领用数量超过剩余量」

   **步骤 5 — 检查是否为管控试剂**
   - 取出试剂瓶的 `cas_number`
   - 调用 `controlled_list_service.get_by_cas_number(cas_number)` 查询管控名录
   - 引用模块：`services/base/controlled_list_service.py` → `ControlledListService.get_by_cas_number()`
   - 如果匹配到管控名录 → 标记 `is_controlled = True`，记录 `controlled_type`（如"易制毒"、"易制爆"）
   - 未匹配 → `is_controlled = False`
   - **管控审批功能暂不实现**，当前仅做检测和提示。后续与人员权限系统一起开发，此处预留接口。

   **步骤 6 — 再次检查状态（防并发）**
   - 重新查询一次试剂瓶状态
   - 如果已变为非「可借」→ 返回失败「该试剂已被其他用户领用」

   **步骤 7 — 创建领用记录**
   - 调用 `id_generator.generate_borrow_record_number()` 生成记录编号
   - 引用模块：`utils/id_generator.py` → `IdGenerator.generate_borrow_record_number()`
   - 组装领用记录数据：记录编号、试剂瓶编号、领用人、领用时间、试剂名称、CAS号、生产日期（后三个从试剂瓶信息继承）
   - 调用 `borrow_record_service.create(borrow_data)` 写入数据库
   - 引用模块：`services/core/borrow_record_service.py` → `BorrowRecordService.create()`
   - 创建失败 → 返回失败「创建领用记录失败」

   **步骤 8 — 更新试剂瓶库存和状态**
   - 计算新剩余量 = 原剩余量 - 领用量
   - 更新字段：剩余量、最后借出时间、可借标记（→「已借出」）、可借判断（→ False）
   - 调用 `reagent_bottle_service.update(bottle.id, updates)` 写入数据库
   - 引用模块：`services/core/reagent_bottle_service.py` → `ReagentBottleService.update()`

   **步骤 9 — 返回结果**
   - 管控试剂 → 返回成功，提示「该试剂为管控化学品(xxx)，请及时补全审批手续」
   - 普通试剂 → 返回成功，提示「领用成功！」

4. 页面汇总所有领用结果：
   - 全部成功 → 显示「全部领用成功！共领用 N 瓶试剂」，清空领用清单
   - 部分成功 → 显示成功数量和失败详情

## 二、试剂归还

### 2.1 入口

页面文件：`pages/3_领用归还.py`
页面内通过 `st.tabs` 分为两个标签页：「试剂领用」和「试剂归还」。
本文档描述「试剂归还」流程。

### 2.2 选择归还试剂

1. 页面调用 `query_service.filter_reagents(status=["已借出", "耗尽"])` 获取所有已借出和耗尽的试剂。
   - 引用模块：`business/query_service.py` → `QueryService.filter_reagents()`
   - 该方法内部调用 `services/core/reagent_bottle_service.py` → `ReagentBottleService.get_all_parsed()` 获取全部试剂瓶数据，然后在 Python 层做状态过滤。
2. 过滤结果以下拉框形式展示，格式为「编号 - 名称 (状态)」。
3. 用户选择要归还的试剂瓶。

### 2.3 填写归还信息

1. **归还人**：
   - 页面调用 `borrow_service.get_latest_borrow_record(bottle_number)` 获取该试剂瓶最近的领用记录。
   - 引用模块：`business/borrow_service.py` → `BorrowService.get_latest_borrow_record()`
   - 如果存在领用记录，默认填入借出人姓名（可修改）。
   - 如果无领用记录，用户手动输入。

2. **归还时剩余量**：
   - 页面提供两种方式输入：
     - 滑块输入：0% - 100%，步长 1%
     - 数值显示：根据百分比和试剂规格自动计算实际剩余量（单位：g）
   - 默认值为 100%（即整瓶归还）。

### 2.4 确认归还

1. 用户点击「确认归还」按钮。
2. 页面调用 `return_service.reagent_return(bottle_number, return_user, remaining_qty)`。
   - 引用模块：`business/return_service.py` → `ReturnService.reagent_return()`

3. **业务层 `reagent_return()` 内部执行以下校验和操作流程**：

   **步骤 1 — 参数校验**
   - 校验 `bottle_number` 为正整数
   - 校验 `return_user` 非空
   - 校验 `remaining_qty` 非负数
   - 任一失败 → 返回 `ServiceResult.fail`

   **步骤 2 — 查询试剂瓶**
   - 调用 `reagent_bottle_service.get_by_bottle_number(bottle_number)` 查询试剂瓶
   - 引用模块：`services/core/reagent_bottle_service.py` → `ReagentBottleService.get_by_bottle_number()`
   - 未找到 → 返回失败「未找到该试剂瓶」

   **步骤 3 — 查找关联的领用记录**
   - 调用 `borrow_record_service.get_by_bottle_number(bottle_number)` 获取该试剂瓶的所有领用记录
   - 引用模块：`services/core/borrow_record_service.py` → `BorrowRecordService.get_by_bottle_number()`
   - 按领用时间倒序排序，取最新一条作为关联记录
   - 记录 `linked_borrow_record_id` 和 `linked_borrow_record_num`（用于后续关联更新）

   **步骤 4 — 生成归还记录编号**
   - 调用 `id_generator.generate_return_record_number()` 生成记录编号
   - 引用模块：`utils/id_generator.py` → `IdGenerator.generate_return_record_number()`

   **步骤 5 — 创建归还记录**
   - 组装归还记录数据：记录编号、试剂瓶编号、归还人、归还时间、剩余量、最后更新时间、修改人
   - 如果存在关联领用记录，添加 `linked_borrow_record_number` 字段
   - 调用 `return_record_service.create(return_data)` 写入数据库
   - 引用模块：`services/core/return_record_service.py` → `ReturnRecordService.create()`
   - 创建失败 → 返回失败「创建归还记录失败」

   **步骤 6 — 更新领用记录的关联字段**
   - 如果存在关联领用记录，更新其 `linked_return_record_number`、`last_update_time`、`modifier` 字段
   - 调用 `borrow_record_service.update(linked_borrow_record_id, borrow_update_data)` 写入数据库
   - 引用模块：`services/core/borrow_record_service.py` → `BorrowRecordService.update()`
   - 更新失败不影响归还流程（仅记录警告日志）

   **步骤 7 — 更新试剂瓶库存和状态**
   - 根据剩余量判断新状态：
     - `remaining_qty > 0` → `borrowable_flag = "可借"`，`borrowable_check = True`
     - `remaining_qty == 0` → `borrowable_flag = "耗尽"`，`borrowable_check = False`
   - 更新字段：剩余量、最后归还时间、最后归还记录号、可借标记、可借判断
   - 调用 `reagent_bottle_service.update(bottle.id, updates)` 写入数据库
   - 引用模块：`services/core/reagent_bottle_service.py` → `ReagentBottleService.update()`

   **步骤 8 — 返回结果**
   - 返回成功，提示「归还成功！」

4. 页面显示归还结果：
   - 成功 → 显示「归还成功！」，刷新页面
   - 失败 → 显示错误信息
