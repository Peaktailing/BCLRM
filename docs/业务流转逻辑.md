# 业务流转逻辑说明

> **开发规范（强制）**：每次更新代码前，必须先阅读本文件。如果业务逻辑描述中有卡住的地方（标注了"待确认"），先向用户确认再继续；如果逻辑通顺，则以本文档为架构去更新代码。

## 一、试剂领用

### 1.1 入口

页面文件：`pages/3_领用归还.py`
页面内通过 `st.tabs` 分为两个标签页：「试剂领用」和「试剂归还」。
本文档仅描述「试剂领用」流程，「试剂归还」见第二节。

### 1.2 搜索与筛选

1. 用户在搜索框中输入关键词（试剂名称、CAS号、编号均可）。
2. 用户可勾选「只显示可借」复选框（默认勾选）。
3. 页面调用 `query_service.filter_reagents(keyword=..., borrowable_only=...)` 进行过滤。
   - 引用模块：`business/query_service.py` → `QueryService.filter_reagents()`
   - 该方法内部调用 `services/core/reagent_bottle_service.py` → `ReagentBottleService.get_all_parsed()` 获取全部试剂瓶数据，然后在 Python 层做关键词匹配和状态过滤。
4. 过滤结果以表格形式展示，包含列：选择（复选框）、编号、名称、CAS号、剩余量、规格、纯度、状态（🟢可借/🔵已借出/🔴耗尽）、存储位置。

> **业务规则确认**：只有 `borrowable_flag = "可借"` 且 `remaining_quantity > 0` 的试剂瓶才可被借出。当前为整瓶借出模式，领用数量等于试剂瓶当前剩余量。

### 1.3 添加到领用清单

1. 用户在表格中勾选一行或多行试剂。
2. 点击「添加 N 瓶试剂到领用清单」按钮。
3. 选中的试剂被追加到 `st.session_state.borrow_cart`（购物车列表），已存在于清单中的不重复添加。
4. 页面下方展示领用清单表格，包含列：序号、编号、名称、CAS号、规格、剩余量、删除。
5. 用户可在清单中删除单条（修改删除列内容）或点击「清空清单」全部清空。

> **交互说明**：当前删除操作通过修改"删除"列内容实现（非🗑️则删除），此交互方式待后续测试优化。

### 1.4 选择领用人

1. 页面调用 `borrow_service.get_all_borrow_users()` 获取历史领用人列表。
   - 引用模块：`business/borrow_service.py` → `BorrowService.get_all_borrow_users()`
   - 该方法内部调用 `services/core/borrow_record_service.py` → `BorrowRecordService.get_all_parsed()` 获取全部领用记录，提取去重后的用户名列表并排序。
2. 用户通过下拉框选择领用人。

### 1.5 确认领用

1. 用户点击「确认领用」按钮。
2. 页面遍历领用清单中的每一瓶试剂，逐瓶调用 `borrow_service.reagent_borrow(bottle_number, user, borrow_qty)`。
   - 引用模块：`business/borrow_service.py` → `BorrowService.reagent_borrow()`
   - `borrow_qty` 取该试剂瓶的当前剩余量（即整瓶借出）。

3. **业务层 `reagent_borrow()` 内部执行以下校验和操作流程**：

   **步骤 1 — 参数校验**
   - 校验 `bottle_number` 为正整数
   - 校验 `user` 非空
   - 校验 `borrow_qty` 大于 0
   - 任一失败 → 返回 `ServiceResult.fail`

   **步骤 2 — 查询试剂瓶**
   - 调用 `reagent_bottle_service.get_by_bottle_number(bottle_number)` 查询试剂瓶
   - 引用模块：`services/core/reagent_bottle_service.py` → `ReagentBottleService.get_by_bottle_number()`
   - 未找到 → 返回失败「未找到该试剂瓶」

   **步骤 3 — 校验试剂瓶状态**
   - 检查 `borrowable_flag` 是否为「可借」
   - 非「可借」→ 返回失败「该试剂当前状态为『xxx』，不可借出」

   **步骤 4 — 校验领用数量**
   - 检查剩余量是否已知（非 None）
   - 检查 `borrow_qty <= remaining_qty`
   - 超出 → 返回失败「领用数量超过剩余量」

   **步骤 5 — 检查是否为管控试剂**
   - 取出试剂瓶的 `cas_number`
   - 调用 `controlled_list_service.get_by_cas_number(cas_number)` 查询管控名录
   - 引用模块：`services/base/controlled_list_service.py` → `ControlledListService.get_by_cas_number()`
   - 如果匹配到管控名录 → 标记 `is_controlled = True`，记录 `controlled_type`（如"易制毒"、"易制爆"）
   - 未匹配 → `is_controlled = False`
   - **管控审批功能暂不实现**，当前仅做检测和提示。后续与人员权限系统一起开发，此处预留接口。

   **步骤 6 — 再次检查状态（防并发）**
   - 重新查询一次试剂瓶状态
   - 如果已变为非「可借」→ 返回失败「该试剂已被其他用户领用」

   **步骤 7 — 创建领用记录**
   - 调用 `id_generator.generate_borrow_record_number()` 生成记录编号
   - 引用模块：`utils/id_generator.py` → `IdGenerator.generate_borrow_record_number()`
   - 组装领用记录数据：记录编号、试剂瓶编号、领用人、领用时间、试剂名称、CAS号、生产日期（后三个从试剂瓶信息继承）
   - 调用 `borrow_record_service.create(borrow_data)` 写入数据库
   - 引用模块：`services/core/borrow_record_service.py` → `BorrowRecordService.create()`
   - 创建失败 → 返回失败「创建领用记录失败」

   **步骤 8 — 更新试剂瓶库存和状态**
   - 计算新剩余量 = 原剩余量 - 领用量
   - 更新字段：剩余量、最后借出时间、可借标记（→「已借出」）、可借判断（→ False）
   - 调用 `reagent_bottle_service.update(bottle.id, updates)` 写入数据库
   - 引用模块：`services/core/reagent_bottle_service.py` → `ReagentBottleService.update()`

   **步骤 9 — 返回结果**
   - 管控试剂 → 返回成功，提示「该试剂为管控化学品(xxx)，请及时补全审批手续」
   - 普通试剂 → 返回成功，提示「领用成功！」

4. 页面汇总所有领用结果：
   - 全部成功 → 显示「全部领用成功！共领用 N 瓶试剂」，清空领用清单
   - 部分成功 → 显示成功数量和失败详情

## 二、试剂归还

### 2.1 入口

页面文件：`pages/3_领用归还.py`
页面内通过 `st.tabs` 分为两个标签页：「试剂领用」和「试剂归还」。
本文档描述「试剂归还」流程。

### 2.2 选择归还试剂

1. 页面调用 `query_service.filter_reagents(status=["已借出", "耗尽"])` 获取所有已借出和耗尽的试剂。
   - 引用模块：`business/query_service.py` → `QueryService.filter_reagents()`
   - 该方法内部调用 `services/core/reagent_bottle_service.py` → `ReagentBottleService.get_all_parsed()` 获取全部试剂瓶数据，然后在 Python 层做状态过滤。
2. 过滤结果以下拉框形式展示，格式为「编号 - 名称 (状态)」。
3. 用户选择要归还的试剂瓶。

### 2.3 填写归还信息

1. **归还人**：
   - 页面调用 `borrow_service.get_latest_borrow_record(bottle_number)` 获取该试剂瓶最近的领用记录。
   - 引用模块：`business/borrow_service.py` → `BorrowService.get_latest_borrow_record()`
   - 如果存在领用记录，默认填入借出人姓名（可修改）。
   - 如果无领用记录，用户手动输入。

2. **归还时剩余量**：
   - 页面提供两种方式输入：
     - 滑块输入：0% - 100%，步长 1%
     - 数值显示：根据百分比和试剂规格自动计算实际剩余量（单位：g）
   - 默认值为 100%（即整瓶归还）。

### 2.4 确认归还

1. 用户点击「确认归还」按钮。
2. 页面调用 `return_service.reagent_return(bottle_number, return_user, remaining_qty)`。
   - 引用模块：`business/return_service.py` → `ReturnService.reagent_return()`

3. **业务层 `reagent_return()` 内部执行以下校验和操作流程**：

   **步骤 1 — 参数校验**
   - 校验 `bottle_number` 为正整数
   - 校验 `return_user` 非空
   - 校验 `remaining_qty` 非负数
   - 任一失败 → 返回 `ServiceResult.fail`

   **步骤 2 — 查询试剂瓶**
   - 调用 `reagent_bottle_service.get_by_bottle_number(bottle_number)` 查询试剂瓶
   - 引用模块：`services/core/reagent_bottle_service.py` → `ReagentBottleService.get_by_bottle_number()`
   - 未找到 → 返回失败「未找到该试剂瓶」

   **步骤 3 — 查找关联的领用记录**
   - 调用 `borrow_record_service.get_by_bottle_number(bottle_number)` 获取该试剂瓶的所有领用记录
   - 引用模块：`services/core/borrow_record_service.py` → `BorrowRecordService.get_by_bottle_number()`
   - 按领用时间倒序排序，取最新一条作为关联记录
   - 记录 `linked_borrow_record_id` 和 `linked_borrow_record_num`（用于后续关联更新）

   **步骤 4 — 生成归还记录编号**
   - 调用 `id_generator.generate_return_record_number()` 生成记录编号
   - 引用模块：`utils/id_generator.py` → `IdGenerator.generate_return_record_number()`

   **步骤 5 — 创建归还记录**
   - 组装归还记录数据：记录编号、试剂瓶编号、归还人、归还时间、剩余量、最后更新时间、修改人
   - 如果存在关联领用记录，添加 `linked_borrow_record_number` 字段
   - 调用 `return_record_service.create(return_data)` 写入数据库
   - 引用模块：`services/core/return_record_service.py` → `ReturnRecordService.create()`
   - 创建失败 → 返回失败「创建归还记录失败」

   **步骤 6 — 更新领用记录的关联字段**
   - 如果存在关联领用记录，更新其 `linked_return_record_number`、`last_update_time`、`modifier` 字段
   - 调用 `borrow_record_service.update(linked_borrow_record_id, borrow_update_data)` 写入数据库
   - 引用模块：`services/core/borrow_record_service.py` → `BorrowRecordService.update()`
   - 更新失败不影响归还流程（仅记录警告日志）

   **步骤 7 — 更新试剂瓶库存和状态**
   - 根据剩余量判断新状态：
     - `remaining_qty > 0` → `borrowable_flag = "可借"`，`borrowable_check = True`
     - `remaining_qty == 0` → `borrowable_flag = "耗尽"`，`borrowable_check = False`
   - 更新字段：剩余量、最后归还时间、最后归还记录号、可借标记、可借判断
   - 调用 `reagent_bottle_service.update(bottle.id, updates)` 写入数据库
   - 引用模块：`services/core/reagent_bottle_service.py` → `ReagentBottleService.update()`

   **步骤 8 — 返回结果**
   - 返回成功，提示「归还成功！」

4. 页面显示归还结果：
   - 成功 → 显示「归还成功！」，刷新页面
   - 失败 → 显示错误信息

## 三、试剂入库

### 3.1 入口

页面文件：`pages/2_试剂入库.py`

### 3.2 加载数据源

页面启动时，从业务层获取以下下拉框数据源：

1. **化学品名称列表**：调用 `inventory_service.get_available_chemical_names()`
   - 引用模块：`business/inventory_service.py` → `InventoryService.get_available_chemical_names()`
   - 该方法内部调用 `services/base/chemical_service.py` → `ChemicalService.get_all_parsed()` 获取全部化学品，提取名称并排序。

2. **供应商列表**：调用 `inventory_service.get_available_suppliers()`
   - 引用模块：`business/inventory_service.py` → `InventoryService.get_available_suppliers()`
   - 该方法内部调用 `services/base/supplier_service.py` → `SupplierService.get_all()` 获取全部供应商。

3. **存储位置列表**：调用 `inventory_service.get_available_storage_locations()`
   - 引用模块：`business/inventory_service.py` → `InventoryService.get_available_storage_locations()`
   - 该方法内部调用 `services/base/storage_location_service.py` → `StorageLocationService.get_all()` 获取全部存储位置。

4. **试剂类型列表**：调用 `inventory_service.get_available_reagent_types()`
   - 引用模块：`business/inventory_service.py` → `InventoryService.get_available_reagent_types()`
   - 该方法内部调用 `services/base/reagent_type_service.py` → `ReagentTypeService.get_all_names()` 获取全部试剂类型。
   - 如果列表为空，使用默认值：`["普通固体试剂", "普通液体试剂", "胶体试剂/培养基", "标准品", "气体钢瓶", "生化试剂"]`

5. **化学品名称到CAS号的映射**：
   - 遍历化学品名称列表，逐个调用 `inventory_service.get_chemical_info_by_name(name)` 获取化学品信息
   - 引用模块：`business/inventory_service.py` → `InventoryService.get_chemical_info_by_name()`
   - 构建 `name_to_cas` 字典，同时支持 `name` 和 `display_name` 作为键

### 3.3 填写入库表单

页面通过 `st.form` 构建入库表单，包含以下字段：

**左列（col1）**：
1. **试剂名称**（必填）：下拉框，从 `chemical_names` 列表中选择
   - 选择后自动匹配 CAS号（只读）
   - 匹配逻辑：先从 `name_to_cas` 字典查找，找不到则调用 `inventory_service.get_chemical_info_by_name()` 查询

2. **CAS号**（只读）：根据试剂名称自动匹配，不可修改

3. **剩余量(g)**（必填）：数字输入框，默认 500.0，最小值 0.001
   - 当前试剂瓶中的实际剩余量

4. **规格(g)**（必填）：数字输入框，默认 500.0，最小值 0.001
   - 试剂瓶的标称规格

**右列（col2）**：
5. **纯度**（必填）：下拉框，选项为 `["分析纯", "化学纯", "优级纯", "色谱纯", "光谱纯", "电子纯", "工业纯"]`，默认选中"分析纯"

6. **试剂类型**（必填）：下拉框，从 `reagent_type_names` 列表中选择，默认选中"普通固体试剂"

7. **采购单价(元)**（可选）：数字输入框，最小值 0.0

8. **供应商**（可选）：下拉框，从 `supplier_names` 列表中选择

9. **生产日期**（可选）：日期选择器

10. **存储位置**（可选）：下拉框，从 `storage_location_names` 列表中选择

**前端校验**：
- 如果 `remaining_qty > spec`，显示警告「剩余量超过规格，可能输入有误」

### 3.4 确认入库

1. 用户点击「确认入库」按钮。
2. 页面进行前端校验：
   - 试剂名称不能为空
   - CAS号不能为空
3. 校验通过后，调用 `inventory_service.create_inventory_record(...)` 提交入库。
   - 引用模块：`business/inventory_service.py` → `InventoryService.create_inventory_record()`

4. **业务层 `create_inventory_record()` 内部执行以下校验和操作流程**：

   **步骤 1 — 校验输入数据**
   - 调用 `self.validate_inventory_inputs(reagent_name, cas_number, remaining_quantity, specification)`
   - 引用方法：`InventoryService.validate_inventory_inputs()`
   - 校验内容：
     - 试剂名称非空
     - CAS号非空
     - 剩余量 > 0
     - 规格 > 0
     - 剩余量 <= 规格
     - 试剂名称必须在化学品信息表中（调用 `chemical_service.get_all_parsed()` 获取全部化学品名称进行比对）
   - 任一失败 → 返回 `ServiceResult.fail`

   **步骤 2 — 生成并校验条码唯一性**
   - 调用 `id_generator.generate_barcode()` 生成条码
   - 引用模块：`utils/id_generator.py` → `IdGenerator.generate_barcode()`
   - 调用 `reagent_bottle_service.get_by_barcode(barcode)` 检查是否冲突
   - 引用模块：`services/core/reagent_bottle_service.py` → `ReagentBottleService.get_by_barcode()`
   - 冲突 → 返回失败「条码生成冲突，请重试」

   **步骤 3 — 检查管控状态**
   - 调用 `self.verify_controlled_status(cas_number)`
   - 引用方法：`InventoryService.verify_controlled_status()`
   - 该方法内部调用 `controlled_list_service.get_by_cas_number(cas_number)` 查询管控名录
   - 引用模块：`services/base/controlled_list_service.py` → `ControlledListService.get_by_cas_number()`
   - 如果匹配到管控名录 → 标记 `is_controlled = True`，记录 `controlled_type`

   **步骤 4 — 生成试剂瓶编号**
   - 调用 `id_generator.generate_bottle_number()` 生成编号
   - 引用模块：`utils/id_generator.py` → `IdGenerator.generate_bottle_number()`

   **步骤 5 — 构建入库数据**
   - 组装试剂瓶数据字典，包含：
     - 必填：试剂瓶编号、条码、试剂名称、CAS号、剩余量、规格、入库日期、可借标记、可借判断
     - 可选：纯度、试剂类型、采购单价、供应商、生产日期、存储位置
   - 可借标记逻辑：`remaining_quantity > 0` → "可借"，否则 → "耗尽"

   **步骤 6 — 创建入库记录**
   - 调用 `reagent_bottle_service.create(inventory_data)` 写入数据库
   - 引用模块：`services/core/reagent_bottle_service.py` → `ReagentBottleService.create()`
   - 创建失败 → 返回失败「入库失败，请检查网络连接」

   **步骤 7 — 返回结果**
   - 管控试剂 → 返回成功，提示「入库成功！试剂瓶编号：xxx，条码：xxx\n注意：该试剂为管控化学品（xxx）」
   - 普通试剂 → 返回成功，提示「入库成功！试剂瓶编号：xxx，条码：xxx」

5. 页面显示入库结果：
   - 成功 → 显示成功信息，刷新页面
   - 失败 → 显示错误信息

## 四、化学品信息管理

### 4.1 入口

页面文件：`pages/7_化学品信息管理.py`

### 4.2 化学品列表展示

1. 页面启动时，调用 `chemical_manage_service.get_all_chemicals()` 获取全部化学品列表。
   - 引用模块：`business/chemical_service.py` → `ChemicalManageService.get_all_chemicals()`
   - 该方法内部调用 `services/base/chemical_service.py` → `ChemicalService.get_all_parsed()` 获取全部化学品。

2. 用户可通过顶部搜索框进行模糊搜索（匹配名称或显示名称）。
   - 调用 `chemical_manage_service.search_chemicals(search_query)` 进行过滤。
   - 引用模块：`business/chemical_service.py` → `ChemicalManageService.search_chemicals()`
